⚡ QUICK WORKFLOW

  1. Verify Health
     GET /health
     └─ Check status, version, active sessions

  2. Send Message
     POST /chat
     └─ Get instant response from AI

  3. Stream Long Response
     GET /chat/stream
     └─ Real-time streaming for lengthy outputs

  4. Retrieve History
     GET /session/{session_id}/messages
     └─ Access full conversation

  5. Clean Up (Optional)
     DELETE /session/{session_id}
     └─ Remove session when done

🚀 GETTING STARTED

  1. Import: postman_collection.json into Postman
  2. Set: Environment variables (base_url, api_key, etc.)
  3. Test: Click Health Check endpoint
  4. Explore: Try Chat endpoints
  5. Debug: Check /docs or /redoc for full API docs

📚 HEADERS & AUTH

  Content-Type: application/json
  Authorization: Bearer {{api_key}}  [optional, for protected APIs]

  For local dev without auth:
    - Leave {{api_key}} empty
    - Disable Authorization header

✅ COMMON TESTS

  • Health Check should return status: "healthy"
  • Chat response includes: response, session_id, message_id
  • Messages endpoint returns array of message objects
  • WebSocket connects successfully to ws://base_url/ws/chat

❓ TROUBLESHOOTING

  "Connection refused"
    → Verify base_url is correct
    → Check API server is running (http://localhost:8000/health)

  "Unauthorized" (401)
    → Set {{api_key}} if authentication required
    → Check Authorization header is enabled

  "WebSocket fails"
    → Use ws:// not http:// for WebSocket URLs
    → Verify session_id is valid

  "Empty response"
    → Check LLM provider configured (GET /health)
    → Verify LLM API key set (if required)
