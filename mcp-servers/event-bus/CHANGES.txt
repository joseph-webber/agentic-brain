Function 1: send_voice_request(prompt, lady="karen", priority="normal")
─────────────────────────────────────────────────────────────────────
Location: server.py, lines 242-278
Purpose: Send LLM request that generates spoken response with lady voice

Signature:
  send_voice_request(prompt: str, lady: str, priority: str) -> dict

Parameters:
  • prompt (str) - Prompt to send to LLM
  • lady (str) - Voice lady name (default: "karen")
  • priority (str) - Request priority (default: "normal")

Returns:
  • request_id - UUID for tracking
  • lady - Selected lady name
  • voice_name - macOS voice name
  • region - Lady's region
  • prompt_preview - First 80 chars of prompt
  • priority - Request priority
  • note - Information about response delivery

Implementation:
  - Validates lady name against VOICE_LADIES (14 options)
  - Fallbacks to "karen" if invalid lady
  - Creates voice_llm_request event
  - Emits to brain.voice.llm topic
  - Includes fallback chain for LLM: [claude, openrouter, emulator]

Function 2: broadcast_voice(message, lady="karen")
────────────────────────────────────────────────────
Location: server.py, lines 281-325
Purpose: Broadcast spoken message with lady voice

Signature:
  broadcast_voice(message: str, lady: str) -> dict

Parameters:
  • message (str) - Message to speak
  • lady (str) - Voice lady name (default: "karen")

Returns:
  • lady - Selected lady name
  • voice_name - macOS voice name
  • region - Lady's region
  • message - The message that was spoken
  • message_length - Length of message
  • note - Information about broadcast

Implementation:
  - Validates lady name against VOICE_LADIES (14 options)
  - Fallbacks to "karen" if invalid lady
  - Creates voice_response event
  - Emits to brain.voice.response topic
  - Executes local macOS TTS: say -v {voice} -r {rate} {message}
  - Handles subprocess errors gracefully (30s timeout)
  - Never breaks even if local TTS unavailable

NEW EVENT TOPICS
================================================================================

1. brain.voice.input
   ──────────────────
   Purpose: User voice input events
   Producers: Voice input capture systems
   Consumers: Voice daemon, analytics
   Format: {type: "voice_input", input_id, text, confidence, timestamp}
   Status: Documented, ready for future use

2. brain.voice.response
   ──────────────────────
   Purpose: Lady voice responses
   Producers: broadcast_voice(), voice system
   Consumers: Voice daemon, monitoring, analytics
   Format: {type: "voice_response", message, lady, voice_name, region, timestamp}
   Status: Active (used by broadcast_voice)

3. brain.voice.conversation
   ─────────────────────────
   Purpose: Conversation state changes
   Producers: Voice conversation manager
   Consumers: Voice daemon, analytics
   Format: {type: "conversation_state", conversation_id, state, current_lady, participants}
   Status: Documented, ready for future use

4. brain.voice.llm
   ────────────────
   Purpose: LLM requests for voice responses
   Producers: send_voice_request()
   Consumers: Voice daemon, LLM services
   Format: {type: "voice_llm_request", request_id, prompt, lady, voice_name, region, priority, fallback_chain}
   Status: Active (used by send_voice_request)
