═══════════════════════════════════════════════════════════════════
METRICS MEASURED
═══════════════════════════════════════════════════════════════════

For each provider test, the following metrics are captured:

1. Time to First Token (TTFT) - milliseconds
   - Measures latency until first response token appears
   - Critical for user experience
   - Target: <100ms local, <150ms Groq, <300ms cloud

2. Total Response Time - milliseconds
   - Complete response generation duration
   - Used for efficiency measurement
   - Depends on response length

3. Throughput - tokens per second
   - Generation speed metric
   - Calculated: token_count / (total_time_ms / 1000)
   - Higher = faster generation

4. Token Count - count
   - Number of tokens in the response
   - Used for throughput calculation
   - Rough estimate: text.length / 4

5. Response Size - bytes
   - Total bytes in response
   - Used for network analysis
   - Correlation with token count

6. Success Status - boolean
   - Whether the request succeeded
   - Includes error messages on failure
   - Used for reliability assessment

7. Target Compliance - boolean
   - Whether TTFT met provider target
   - Shows performance goal achievement
   - 100% = all targets met
